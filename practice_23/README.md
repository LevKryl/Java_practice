# Перегрузка и переопределение

## Практика 1: перегрузка (overloading)

Одинаковое имя, разные параметры.

Calculator {
    int add(int a, int b)
    int add(int a, int b, int c)
    double add(double a, double b)
}

## Практика 2: переопределение (overriding)

Метод подкласса с таким же именем и параметрами как в суперклассе.

Если не совпадает тип возвращаемого значения - ошибка компиляции.

## Практика 3: аннотация @Override

Помогает обнаружить ошибку, когда метод не переопределяет суперкласс.

Примеры ошибок, которые ловит @Override:
- Опечатка в имени метода
- Неправильные параметры
- Неправильный тип возвращаемого значения

Если написать @Override, но метод на самом деле не переопределяет суперкласс - компилятор выдаст ошибку.
