# Логические блоки в Java

## Что выведет код

static logic
logic (1) id= 0
logic (2) id= 1
ctor id= 100

## Почему так

1. static logic - статический блок вызывается при загрузке класса (первым)

2. logic (1) id= 0 - нестатический блок вызывается до конструктора, поле id ещё не инициализировано (значение по умолчанию 0)

3. logic (2) id= 1 - после явной инициализации поля id = 1

4. ctor id= 100 - конструктор выполняется последним

## Порядок выполнения при создании объекта

1. Статические блоки (один раз при загрузке класса)
2. Нестатические блоки (в порядке написания)
3. Явная инициализация полей
4. Конструктор
